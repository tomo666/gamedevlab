# ステートメント/変数/型

## 目的

- ステートメントを理解する
- プログラムのコメントについて理解する
- 変数と型を理解する
- JBScriptで使える変数（型）の種類を知る

---

## ステートメントとは

ステートメントは、プログラムの中の **「1つの命令」** です。ゲームは複数の命令を順番に書いてプログラムを実行していきます。

たとえば、次のプログラムは「hp という箱を作って、10 を入れる」という2つの命令から構成されています。

```csharp
int hp;
hp = 10;
```

JBScriptでは、多くの命令は行末に **`;`（セミコロン）** を付けて書きます。

- セミコロンは「この命令はここで終わり！」という区切り
- 基本的には、1行に1つずつ命令を書いてプログラムを構築していきます

---

## ステートメントの例

ステートメントはいろいろあります。たとえばこんな感じです。

**1. 変数を作る（宣言）**

```csharp
int coins;
```

**2. 値を入れる（代入）**

```csharp
coins = 0;
```

**3. 計算して入れる（代入の一種）**

```csharp
coins = coins + 1;
```

**4. 条件で分ける（if文）**

```csharp
if(coins >= 100) {
  // 100枚以上なら1up!
}
```

**5. 機能を呼び出す（関数呼び出し）**

```csharp
Common.SetScreenScale(5f);
```

**6. これは「式」で、単体では命令にならない**

次のようなものは **計算（式）** ですが、これだけだと「何もしない」ので、ふつうはステートメントとして書きません。

```csharp
coins + 1
```

「計算した結果をどこかに入れる」など、命令の形にして使います。

```csharp
coins = coins + 1;
```

## コメント

コメントは、プログラムの中に書ける**メモ**です。
コメント自体はプログラムとして実行されないので、読んだ人（未来の自分も含む）がプログラムの内容を説明・把握するするために使います。

JBScriptには、次の2種類のコメントの書き方があります。

**1. 1行コメント（//）**

`//` の右側は、その行の最後までがコメント扱いになります。

```csharp
// HPの初期値
int hp = 10;
int mp = 5; // MPの初期値
```

**2. 複数行コメント（/\* \*/）**

`/*` と `*/` で囲んだ部分は、複数行まとめてコメントにできます。

```csharp
/*
  ここはコメント。
  何行書いてもOK。
*/
```

## 変数とは

変数は、ひとことで言うと **「データを入れておく箱」** です。

- プレイヤーのHP
- 持っているコインの数
- イベントフラグ

こういう情報を、ゲームの中で記憶しておくために使います。

---

## 変数の基本的な使い方

**1. 変数を作る（宣言）**

```csharp
int hp;
```

この行で「hp という箱を用意します」と宣言しています。
この時点では、まだ中身（数）は入っていません。

**2. 値を入れる（代入）**

```csharp
hp = 10;
```

この行で、hp の箱に **10** を入れています。
`=` は「右の値を左の箱に入れる」という意味です。

**3. 使う**

```csharp
if(hp <= 0) {
  // ここでゲームオーバー処理を実装する
}
```

ここでは hp の中身をチェックしています。
もし hp が 0 以下になったら（つまりHPがなくなったら）、ゲームオーバー処理をします。ゲームオーバー処理については未実装なので将来実装するよ、とコメントを入れています。

:::tip
## 宣言と同時に代入もできる

上記の例では一行ずつ宣言と代入をしていますが、変数は宣言と同時に値の代入を一行で記述することもできます。

```csharp
int hp = 10;
bool isAlive = true;
byte coins = 3;
```
:::
---

## チュートリアル: 数を数えてみよう

例えば、プレイヤーがコインを拾ったら、所持しているコインの数を増やしたいですよね。

```csharp
int coins = 0;

// コインを拾った
coins = coins + 1;

// さらに拾った
coins = coins + 1;
```

`coins = coins + 1;` は「今の coins に 1 を足して、もう一度 coins に入れる」という意味です。

## JBScriptで使える変数（型）

JBScriptには、用途に応じていろいろな型（箱のサイズ）があります。以下は変数の型の一覧です。型名、サイズ、代入できる数値の範囲の順で示しています。
ただし、JBScriptを使った実際のゲームプログラムでは `bool` / `int` / `float` 以外はあまり使用しませんので、とりあえずこの3つの型だけ覚えておけばOKです。

### 真偽値（true / false）

- `bool`（8bit）: **true（真） / false（偽）** のどちらかだけを入れられる型です。

「スイッチ」や「フラグ」に向いています。

```csharp
bool hasKey = false; // まだカギを持っていない

// カギを拾った
hasKey = true;

// カギを持っているならドアを開ける
if(hasKey) {
  // ドアを開ける処理
}
```

### 整数（マイナス値も扱える）

- `sbyte`（8bit）: -128 〜 127
- `short`（16bit）: -32,768 〜 32,767
- `int`（32bit）: -2,147,483,648 〜 2,147,483,647
- `long` ※デフォルトは`int`と同じです（後述）

### 整数（マイナス値は扱えない）

- `byte`（8bit）: 0 〜 255
- `ushort`（16bit）: 0 〜 65,535
- `uint`（32bit）: 0 〜 4,294,967,295
- `ulong` ※デフォルトは`uint`と同じです（後述）

### 実数（小数）

- `float`（32bit）: 小数（デフォルトでは **-21.47483647 〜 21.47483647** の範囲）

重要：float型の変数に値を代入する場合は、値の後ろに必ず「f」という文字を付ける必要があります。

例：
```csharp
float alpha = 0.1f;
```

※上級者向け情報：この小数の扱える範囲（精度）は、JBScriptのコンパイル設定（DecimalPointPrecision など）や、JBGEエンジン側の設定で変更できるようにしてあります。

### 列挙型（enum）

`enum`（列挙型）は、**決まった選択肢の中から1つを選ぶ**ための型です。

たとえば「向き（上/下/左/右）」や「状態（待機/歩く/攻撃）」のように、数値に意味を持たせたいときに使います。

```csharp
// 方向を表すenum
public enum Direction {
  Up,
  Down,
  Left,
  Right
}

Direction dir = Direction.Up;

if(dir == Direction.Left) {
  // 左を向いているときの処理
}
```


`enum` の中身は、内部的には **整数（int）** として扱われます。
そのため `==` で比較したり、変数として保存したりするのに向いています。

#### enumは「自動で番号が増える」

`enum` は、先頭（または途中）で番号（数値）を指定すると、**次の項目は自動で +1 ずつ増えていく**という便利な書き方ができます。

たとえば次の例では、`Up` を **5** にすると、
その次の `Down` は **6**、さらに次は **7**…というように自動で番号が振られます。

```csharp
public enum Direction {
  Up = 5,
  Down,
  Left,
  Right
}
```

この仕組みを使うと、
「番号の管理が必要だけど、毎回ぜんぶ手で書くのは面倒！」という場面で助かります。


### null（ヌル）

`null` は **「何も入っていない」** という特別な値（リテラル）です。

- **型ではありません**（`int` や `bool` のような“箱の種類”ではなく、“中身の値”）
- JBStudio / JBGE内部では `null = 0` として扱われます（`false` も 0、`true` は 1）

なので、JBScriptでは `null` は次のように **「0 と同じ」** 感覚で使えます。

```csharp
object p = null; // 参照がまだ無い（= 0）

if(p == null) {
  // まだ何も入っていないときの処理
}
```

注意：`null` が 0 と同じ扱いになるのはJBScriptの仕様です。
他のプログラミング言語の `null` / `nil` と完全に同じ意味とは限りません。


### その他

- `object`（32bit）: オブジェクト参照（主に仮想マシンメモリのアドレスを入れるために使われます）
- `var`（自動）: 代入した値に合わせて、8/16/32bitのいずれかに自動で割り当てられます（ただし上限は後述する `MaxBitSize` に依存します）

:::info 文字列と特殊な型について
- JBScriptには従来のプログラム言語に実装されているような文字列型がありません。`string` / `char`型は用意されていますが、C#で扱うような使い方はできません。（stringはコンパイル時にchar配列に分解されますので、charポインタのように使用することは可能です）。JBScriptではクセが強い場面があるため、実際の制作では文字列を扱いやすくした **`JString`** オブジェクトを使用して文字列操作をします。
- `double`（64bit）/ `decimal`（128bit）も使えますが、重くなりやすいので **基本は使いません**（必要なときだけ）。
- ※補足：JBGEは現在 `MaxBitSize = 32` で運用しているため、`long` や `decimal` を「本来の意味のサイズ」で使うことはできません。
:::


:::tip 上級者向け：MaxBitSize（整数のビット数）
JBScriptは「コンパイル設定の `MaxBitSize`」によって、扱える整数のビット数が変わります。

- JBGEでは現在 `MaxBitSize = 32` 固定で運用しています
- `object` / `long` / `ulong` は、JBGEの既定設定では実質 32bit 扱いになります
- `var` は値に合わせて 8/16/32bit に自動で決まります（ただし最大は `MaxBitSize` まで）
- `decimal`（128bit）は、`MaxBitSize = 128` にしない限り事実上使えません

- 例：MaxBitSize = 64 でコンパイルした場合、`long`（64bit）: -9,223,372,036,854,775,808 〜 9,223,372,036,854,775,807
- 例：MaxBitSize = 64 でコンパイルした場合、`ulong`（64bit）: 0 〜 18,446,744,073,709,551,615

将来的に `MaxBitSize` を見直す可能性があるので、ここは「上級者向けメモ」として残しておきます。
:::
